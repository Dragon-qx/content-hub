# ContentHub — 需求文档（PRD）

> 版本: 1.0 | 创建: 2026-07-15 | Owner: Master

---

## 1. 产品概述

**ContentHub** 是一个工业级全自动化内容管理平台，用于统一管理多个社交账号的内容创作、发布、数据监测和分析。它面向内容团队、运营个人、MCN 机构，提供从内容生产到数据复盘的全链路工具。

---

## 2. 核心目标

| 目标 | 说明 |
|------|------|
| **一站式管理** | 一个平台管理所有社交账号，不再来回切换 |
| **自动化发布** | 定时发布、批量发布、条件触发发布 |
| **数据驱动** | 核心指标一目了然，跨平台对比分析 |
| **团队协作** | 内容审批流程、权限管理、多人协作 |
| **可扩展** | 新平台接入成本低，插件化架构 |

---

## 3. 目标用户

| 角色 | 需求 |
|------|------|
| **运营个人** | 管自己所有账号，高效发布，看数据 |
| **内容团队** | 分工协作，主编审批，小编执行 |
| **MCN 机构** | 批量管理达人和品牌账号，统一调度 |

## 4. 功能范围

### 4.1 账号管理（Account Hub）

- **多平台账号绑定**：支持 OAuth2 / Cookie / API Key 方式绑定以下平台：
  - 微信公众号（文章 + 图文）
  - 微信视频号
  - 抖音（视频 + 图文）
  - 小红书（笔记）
  - Bilibili（视频 + 动态）
  - 微博
  - Twitter/X（推文 + 线程）
  - YouTube（视频 + Shorts + 社区帖）
- **账号分组**：按项目、品牌、平台、运营分组管理
- **权限等级**：Owner / Admin / Editor / Viewer 四级
- **健康度监测**：检测账号状态（是否掉封、API 是否过期）

### 4.2 内容管理（Content Studio）

- **内容创建**：
  - 富文本编辑器（Markdown + 所见即所得）
  - 视频素材管理（上传、转码、封面裁剪）
  - 图片素材库（在线裁剪、滤镜、加水印）
  - AI 辅助写作（标题优化、标签推荐、文案变体）
- **内容库**：所有内容版本化管理，支持标签 / 分类 / 全文搜索
- **内容日历**：日历视图展示所有计划发布内容
- **模板系统**：可复用的内容模板（早安帖、好物推荐、节日营销）

### 4.3 发布调度（Auto Publish）

- **定时发布**：精确到秒，支持多时区
- **批量发布**：一次编辑 → 多平台适配 → 一键分发
- **智能排期**：根据历史数据推荐最佳发布时间
- **发布队列**：可视化队列，支持优先级调整、暂停 / 恢复
- **发布回执**：自动抓取发布状态、链接、截图留档
- **失败重试**：指数退避重试策略 + 失败告警
- **内容适配**：同一内容自动适配各平台格式（字数限制、图片尺寸、视频规格）

### 4.4 数据看板（Analytics Dashboard）

- **核心指标**：
  - 曝光量 / 播放量 / 阅读量
  - 互动量（点赞 / 评论 / 转发 / 收藏）
  - 粉丝净增 / 粉丝画像
  - 变现数据（如有接入）
- **跨平台对比**：同一内容在不同平台表现横向对比
- **趋势分析**：日 / 周 / 月维度趋势图
- **内容表现排行**：Top 内容、最差内容自动标记
- **异常检测**：数据突降 / 突增自动告警
- **自定义报表**：拖拽生成自定义数据报表
- **数据导出**：CSV / Excel / PDF

### 4.5 互动管理（Engagement Hub）

- **评论聚合**：各平台评论统一收件箱
- **快捷回复**：预设回复模板 + AI 辅助生成
- **舆情监控**：关键词监控、情感分析
- **私信管理**：跨平台私信聚合

### 4.6 系统管理（Admin）

- **用户管理**：邀请、禁用、权限调整
- **审批流程**：自定义审批流（一级 / 多级）
- **操作日志**：全审计日志
- **API 管理**：API Key 生成、调用统计、限流
- **存储管理**：素材存储配额、CDN 状态
- **系统配置**：邮件 / 短信 / Webhook 通知渠道

## 5. 非功能性需求

### 5.1 性能

| 指标 | 要求 |
|------|------|
| API P99 延迟 | ≤ 200ms（数据查询 ≤ 500ms） |
| 并发用户 | ≥ 1,000 |
| 发布调度精度 | ± 1 秒 |
| 数据新鲜度 | 核心指标延迟 ≤ 5 分钟 |
| 文件上传 | 支持 ≤ 2GB 单文件 |

### 5.2 可用性

- 系统可用性 ≥ 99.9%
- 支持滚动更新，无停机发布
- 关键服务多副本部署

### 5.3 安全

- 数据传输 TLS 1.3
- 密码 bcrypt / argon2 哈希
- API Key 加密存储
- RBAC 权限模型
- 操作审计日志
- 登录 MFA 支持
- SQL 注入 / XSS / CSRF 防护

### 5.4 可扩展性

- 微服务架构，服务独立开发、部署、伸缩
- 消息队列驱动核心流程
- 插件化平台接入层

## 7. MVP 范围（V1.0）

以下为第一期必须交付的功能：

- 账号管理（绑定 ≥ 3 个平台）
- 内容创建与编辑
- 定时发布
- 核心数据看板（阅读量 / 播放量 / 互动量 / 粉丝）
- 用户系统与权限
- 操作日志

### 不在 V1.0 范围内

- 互动管理（V1.1）
- AI 辅助写作（V1.1）
- 高级自定义报表（V2.0）
- 移动端 App（未来）

## 8. 数据模型（核心实体）

```
User ──┬── Team ──┬── Account (社交账号)
       │         └── Member (团队成员)
       ├── Content ──┬── MediaAsset
       │             ├── PlatformPost (各平台发布记录)
       │             └── ContentTag
       ├── PublishJob (发布任务)
       ├── AnalyticsData (数据快照)
       └── Workflow (审批流)
```

---

## 9. 开放问题

1. **各平台 API 限制**：请求频率限额、字段限制需在技术方案中逐一确认。
2. **合规性**：部分平台（微信公众号）需认证服务号才开放 API。
3. **国际化**：V1.0 仅支持中文界面，后续再做 i18n。
4. **计费模型**：如果部署为 SaaS，需考虑按账号数 / 发布量计费。
